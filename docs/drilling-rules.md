# ドリル加工ロジック抽出（logic-v2.js / blocks-v2.js / gui-v2.js より）

> スコープ: **ドリル加工そのものの分岐ルールだけ**を対象にする。
> 内径バイト（ボーリングバー）による平底仕上げ・奥バイト面取りなど「内径加工バイトが前提」の加工方法
> （例: M40 / M22 / M18 / M15 / G78・M42X3系・トメセン系・G12B・M12のNormal/バイト仕上げ）は、
> ドリルは単なる下穴でありユーザーがG74/G1を選べて特に矛盾がないため、本ドキュメントでは扱わない。
>
> 経緯: UIには常に「ドリルモード（G74/G1）」の選択肢が出るが、一部ワーク種別ではロジック側で
> その選択が無視されて固定動作になっていた。この違和感を解消するため、2026-07-18にUI側で
> 「選択できるケース／固定のケース」を出し分ける改修を実施した（gui-v2.js）。本ドキュメントは
> その固定判定の根拠になっているコード分岐を整理したもの。
>
> 抽出元:
> - `Gコードジェネレータ/assets/logic-v2.js`（ドリルブロック選択チェーン・ドリル深さ計算）
> - `Gコードジェネレータ/assets/blocks-v2.js`（Gコードブロック本体）
> - `Gコードジェネレータ/assets/gui-v2.js`（UI側の固定判定・プロファイル解決）

## 1. ドリルモードが「固定」されるワーク種別・条件

`logic-v2.js` 866〜878行目のドリルブロック選択チェーン。**上から順に該当した条件で確定する。**

```
① usesG18DrillShiageG1Block(workType)                         → G1固定（単動）
   対象: G18_40 / G18_42 / G18_62 / G18_655 / G18_6175（+MH全て）

② isJM8ASWDWorkType(workType) または isM8WorkType(workType)     → 10mmステップ送り固定
   対象: J_M8_300 / J_M8_200（次世代ASWD系）、M8_21 / M8_31（HSS系）

③ (workType===M12||M12_MH) かつ m12FinishType==="hss"           → 10mmステップ送り固定

④ それ以外（M12のNormal/バイト/HGDR含む、内径加工バイト前提のワーク全般）
                                                                 → ユーザーが選んだ drillMode（G74/G1）が有効
```

①②③が「今回UIを固定表示に変えた対象」。④は本ドキュメントの対象外（前提を参照）。

### ①のドリルブロック中身: `getDrillShiageHGDRBlock(depth, "G1")`（`blocks-v2.js` 51〜97行目）

単動1発切削のみ。`G1Z-depth F.15` → `G4U.3`（ドウェル） → `Z1.F2.5`（引き戻し）。ペックサイクル（G74）は使わない。

### ②③のドリルブロック中身: `getDrillShiage10mmStepBlock(depth)`（`blocks-v2.js` 108〜129行目）

G1単動固定、10mm刻みで「切削→Z30.まで引き戻し→次ステップ手前まで早送り」を繰り返し、最終深さのみドウェル(`G4U.3`)を入れて終了。**G74は一切使わない。ステップ量は常に10mm固定（ドリル径によって変わらない）。**

参考: ④のグループが使う `getDrillShiageHGDRBlock` のG74モードは、深さ30mm以下なら一発切込み、30mm超なら「最初30mmをG74（Q3000）→以降10mmステップ」という深さ依存の分岐がある。①②③にはこの深さ分岐は存在しない（常に同じ動き）。

## 2. 一文字ドリル（Ichimonji）との関係

内径スタイル `Ichimonji`（一文字DR平底）を選ぶと、上記①〜④のドリルブロックとは別に、専用工具「一文字ドリル」を使うブロックが追加で走る（`logic-v2.js` 718〜730行目）。

| 条件 | 使用ブロック | 内容 |
|---|---|---|
| style="Ichimonji" かつ (M12/M12_MH または M8系) | `getIchimonjiHirazokoBlock(内径深さ)` | 内径深さ基準（Depth-2.0〜Depth+0.2）で平底仕上げ |
| style="Ichimonji" かつ 上記以外の全ワーク | `getIchimonjiBlock(CP)` | CP±2mmで貫通させる面取り |

- M12/M8のIchimonjiは、①〜③のメインドリル（下穴）＋一文字ドリルによる仕上げ、の2段構成。
- M12はIchimonji時にHSS/HGDRをユーザーが選べる（`gui-v2.js` `resolveM12Profile()`）。**HSS選択時のみ③に該当して10mmステップ固定になり、HGDR選択時は④（G74/G1選択可能）に落ちる。**
- M8系は選択肢なし固定（`m8Profile` は常に `"drill_ichi_men"`、コード上のコメントより「M8 交差穴: HSSドリル + 一文字面取り（固定）」）。

## 3. ドリル径・ドリル深さの自動計算ルール

`DRILL_DIA_MAP`（`logic-v2.js` 148〜186行目）— 固定対象ワークのみ抜粋:

| ワーク種別 | ドリル径 (mm) |
|---|---|
| G18_40 / _MH | 4.05 |
| G18_42 / _MH | 4.15 |
| G18_62 / _MH | 4.15 |
| G18_655 / _MH | 4.15 |
| G18_6175 / _MH | 4.15 |
| M8_21 | 2.1（ワーク寸法公差 2.1±0.2） |
| M8_31 | 3.0（ワーク寸法公差 3.1±0.2） |
| J_M8_300 | 3.0（ワーク寸法公差 3±0.1） |
| J_M8_200 | 2.0（ワーク寸法公差 2±0.1） |
| M12 / M12_MH | 4.05 |

※ 実際のドリル加工径は公称径より約0.05mm小さくなる（現場からの指摘により2026-07-18訂正: M8_21は2.2→2.1、M8_31は3.2→3.0に修正）。

ASWDドリル肩高さ `ASWD_SHOULDER_MM`（ドリル公称径キー）: `4.0→0.96` / `3.0→0.70` / `2.1→0.50` / `2.0→0.50`

自動計算式（`logic-v2.js` 680〜716行目、ドリル関連のみ抜粋）:

```
style が Hirazoko または Ichimonji かつ 内径深さ入力あり:
    ドリル深さ = 内径深さ + 0.1

style = YoseRelay:
    ドリル深さ = 対ヨセ長さ + 0.3×ドリル径

workType が ASWD系（J_M8_300/200）:
    ドリル深さ = ASWD_SHOULDER_MM[ドリル径] + CP + 1　（ユーザー入力を上書き）
```

## 4. 今回実施したUI改修（gui-v2.js）

- `isDrillModeFixed(workType)`: 上記①②③の条件を判定
- 固定対象のワーク種別を選んでいる間は、ドリルモードのG74/G1選択カードの代わりに「固定」バッジ付きの読み取り専用表示を出す
- M12でIchimonji×HSS/HGDRを切り替えると、固定表示⇔選択カードがその場で切り替わる

表示文言（「G1（単動）固定」「10mmステップ送り固定」）は暫定。現場で使う言葉に合わせて後で調整する。

## 5. ドリル分岐を正しくするための確認事項（質問リスト）

ドリル加工のロジックだけに絞った、業務ルール確認用の質問です。回答をもらい次第、コード・UI・本ドキュメントに反映します。

1. **G18系（φ4.0/4.2/6.2/6.55/6.175）は深さに関わらず常にG1単動1発固定でよいか？** 深い穴（例: 30mm超）でもペック（G74）なしで問題ない認識か。
2. **M8系（φ2.1/3.1）・次世代ASWD系（φ2.0/3.0）は常に10mmステップ送り固定でよいか？** 今後さらに細い/太いドリル径の派生品が増えても同じ運用でよいか。
3. **M12は「HSSドリル選択時のみ10mmステップ固定、HGDRドリル選択時はG74/G1を選べる」という区別は正しいか？** HGDRドリルの方が単動・ペックどちらでも安全に使えるので選択式にしている、という理解で合っているか。
4. **10mmという送りステップ量は、ドリル径や機種に関わらず常に一律でよいか？**（細いドリルほど折れやすいので本来はもっと短いステップにすべき、等の実務上の配慮の有無）
5. **ASWD系のドリル深さ自動計算式「肩高さ＋CP＋1」と、`ASWD_SHOULDER_MM`の値（φ4.0→0.96 / φ3.0→0.70 / φ2.1→0.50 / φ2.0→0.50）は現行のドリル仕様と一致しているか？**
6. **一文字ドリル（Ichimonji）によるドリル深さ+0.1mmという余裕代（内径深さ+0.1）は今も正しい値か？**
7. **今回「固定」表示にした3グループ（G18系／M8・ASWD系／M12-HSS）以外に、実は固定にすべきワーク種別・条件が漏れていないか？**（逆に、固定にしたグループの中に本来選択可能であるべきものが混ざっていないか）
